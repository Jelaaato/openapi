Authentication

OAuth is the main authentication and authorization framework used for hopprLab's OpenAPI. To know and read more about OAuth, please refer to RFC 6749



Access Token

An access token is the credential that can be used to access an API. It is unique to a user and should be stored securely and should be included to every request sent to the server.
To retrieve an access token, please refer to the instructions below.

1. Send a POST request to the endpoint, https://hopprlab.com/openapi/token together with the needed credentials.

{ "grant_type" : "password", "username" : "SampleClient", "password" : "samplepassword", "client_id" : "TestClient", "client_secret" : "TestSecret" }



2. Once the request is successful, an Access and Refresh Token will be sent by the authorization server.

3. The Access Token must be included in the Authorization header for every request that will be sent to the resource server.

4. Use the Refresh Token to retrieve a new Access Token once it expires.

Moreover, access token expires after an hour while refresh tokens last for 24 hours and for every request to be sent, client_id and client_secret are required.
